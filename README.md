# OverIt – Student Burnout Monitoring & Wellness Recommendation System

## Overview

OverIt is a web-based student wellness monitoring system designed to identify burnout risks among students through periodic surveys, analyze burnout levels using weighted behavioral indicators, and provide personalized wellness recommendations.

The system allows students to regularly assess their mental well-being while enabling administrators to monitor burnout trends, generate analytical reports, identify high-risk students, and take preventive actions through a centralized dashboard.

The primary objective of OverIt is to promote student well-being by providing data-driven insights into academic stress, workload management, sleep patterns, and overall mental health.

## Burnout Analysis Methodology

The system calculates burnout scores using a weighted analytical model based on multiple wellness indicators collected through surveys.

Factors considered include:

* Sleep Quality
* Stress Level
* Academic Pressure
* Assignment Load
* Deadline Proximity
* Physical Activity
* Social Interaction
* Break Frequency
* Mood Level

Each parameter contributes to the final burnout score according to predefined weights. The score is normalized on a scale of 0–100 and categorized into:

| Burnout Score | Risk Level    |
| ------------- | ------------- |
| 0 – 39        | Low Risk      |
| 40 – 69       | Moderate Risk |
| 70 – 100      | Severe Risk   |

## Database Design

### Tables Used

#### student

Stores student profile and authentication information.

#### admin

Stores administrator credentials and management information.

#### survey_submission

Stores weekly survey responses submitted by students.

#### burnout_score

Stores calculated burnout scores and risk categories.

#### recommendation

Stores wellness recommendations generated by the system.

#### risk_alert

Stores risk notifications generated for at-risk students.

#### analytics_report

Stores generated analytics reports and statistical summaries.

## Architecture

The application follows the MVC Architecture.

### Model Layer

Contains Java classes representing business entities and data objects.

### View Layer

Implemented using JSP, HTML, CSS, and JavaScript.

### Controller Layer

Implemented using Servlets that handle requests, process business logic, and communicate with the database through DAO classes.

### DAO Layer

Responsible for all database operations, queries, inserts, updates, and data retrieval.

### Utility Layer

Provides reusable helper classes such as database connection management.
